# tanapage — シェア型書店の棚ページ

棚に貼った QR コードから、購入者が「買いました」を報告し、棚主からの返礼メッセージを受け取る。それだけの小さな仕組みです。シェア型書店（一箱本棚オーナー制の本屋）の棚主が、自分の棚に「ページ」を持てるようにします。

- **棚ページ** — 陳列中の本と棚主の一言、「棚主より」のメッセージ。購入報告を送ると返礼のことばが返ってくる
- **棚主管理ページ** — 棚キーでひらく。購入報告の承認、在庫の追加・編集、メッセージの編集、月別の販売履歴

## 設計原則

- **購入者の個人情報はゼロ** — 名前・連絡先の入力欄そのものを作らない。IP や Cookie も記録しない
- **返礼メッセージが体験の山場** — 「買った報告」への見返りは、割引でもポイントでもなく、棚主からのことば
- **EC 的な演出はしない** — カート・在庫数・「残りわずか」の類いは持ち込まない
- **手動でも回る** — すべてのデータはスプレッドシートにあり、シート直編集がいつでも効く

## 構成

```
GitHub Pages (index.html / admin.html)
        │  fetch (JSON)
        ▼
Google Apps Script (Code.gs) ── Google スプレッドシート (books / reports / shelf_config)
```

サーバー費用はかかりません。Google アカウントと GitHub アカウントがあれば動きます。

## 利用形態と読むもの

| 形態 | 想定 | 読む手順書 |
|---|---|---|
| 多テナント型 | 店の運営者が複数の棚主に棚ページを提供する | 運営者: [docs/setup.md](docs/setup.md) + [docs/operator.md](docs/operator.md)<br>各棚主: [docs/shelf-owner.md](docs/shelf-owner.md) |
| 間借り一人型 | シェア型書店に棚を借りている個人が、自分の棚のためだけに立てる | [docs/setup.md](docs/setup.md) → [docs/shelf-owner.md](docs/shelf-owner.md) |
| 個人型 | 自宅の前・イベント・無人販売など、自分の棚に | 同上 |

セットアップの所要時間はおよそ 20 分です。

## ファイル

| ファイル | 役割 |
|---|---|
| `index.html` | 棚ページ（購入者が見る側）。`?s=棚ID` で棚を指定 |
| `admin.html` | 棚主管理ページ。`admin.html?s=棚ID` |
| `Code.gs` | バックエンド。Google Apps Script に貼り付けて使う |
| `docs/` | セットアップ手順・運営者手順・棚主ガイド |
