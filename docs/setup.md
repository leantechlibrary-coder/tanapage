# セットアップ手順（デプロイする人向け）

所要時間の目安: 20分。Google アカウントと、公開用の GitHub リポジトリがあれば足ります。

多テナント型なら店の運営者が、間借り一人型・個人型なら本人が、この手順を一度だけやります。

## 1. スプレッドシート＋GAS

1. Google ドライブで新規スプレッドシートを作成（名前は任意。例:「棚ページDB」）
2. メニュー **拡張機能 → Apps Script** を開く
3. エディタの内容を全部消して、このリポジトリの **Code.gs** を貼り付け、保存
4. 関数ドロップダウンから **setup** を選び、実行
   - 初回は権限承認を求められる。「このアプリはGoogleで確認されていません」→「詳細」→「（プロジェクト名）に移動」で承認
   - 実行後、シートに books / reports / shelf_config の3枚（ヘッダのみ）ができていれば成功
   - 見本データ（ことづて棚 shelf-2f と3冊）で動きを眺めたいときは、代わりに **setupDemo** を実行（試したあとは行ごと削除してよい）
5. **デプロイ → 新しいデプロイ → 種類: ウェブアプリ**
   - 説明: 任意
   - 実行ユーザー: **自分**
   - アクセスできるユーザー: **全員**
6. 表示された **ウェブアプリのURL**（`https://script.google.com/macros/s/…/exec`）をコピー

### 動作確認（この時点でできる）

ブラウザで以下を開いて、JSON が返れば成功:

```
（ウェブアプリのURL）?action=getShelf&shelf=shelf-2f
```

- setup（空）で始めた場合は `{"ok":false,"error":"shelf not found"}` でOK（API疎通の確認になる）
- setupDemo で始めた場合は見本の棚データが返る

## 2. フロントエンド

1. **index.html** と **admin.html** をテキストエディタで開き、上のほうにある

   ```javascript
   const GAS_URL = '…';
   ```

   の中身を、コピーした自分のウェブアプリのURLに差し替える（2ファイルとも同じURL）
2. 自分の GitHub リポジトリに index.html と admin.html を置き、**Settings → Pages** で GitHub Pages を有効化
3. `https://（ユーザー名）.github.io/（リポジトリ名）/?s=（棚ID）` が棚ページのURLになる

※ GAS_URL が空のままなら見本データで動くので、デプロイ前のデザイン確認はローカルでできます。

## 3. 棚をつくる

利用形態によって最初の棚のつくり方が変わります。

### 間借り一人型・個人型（棚は自分の1つだけ）

シートは触らなくてよい。`admin.html?s=（好きな棚ID）` を開くと、棚がまだ1つもないときは**「棚をつくる」画面**になる。棚ID・棚の名前・自己紹介・棚キーを入れれば運用開始。

- 棚IDは半角英数字・ハイフン・アンダースコアで32文字まで
- 棚IDはURLとQRコードに焼き込まれるので、**あとから変えない前提**で決める（借りている棚の番号や屋号のローマ字などが自然）

### 多テナント型（運営者が複数の棚主に割り当てる）

運営者が shelf_config シートに1棚1行を追加する。詳細は [operator.md](operator.md) を参照。

- shelf_id は店の既存の棚番号・記号に合わせる
- **shelf_key_hash は空のまま**にしておく → 各棚主が `admin.html?s=棚ID` を開くと棚キーの設定画面になり、自分で合言葉を決めて使い始める

### 2つ目以降の棚（どの形態でも）

ZINE棚の追加なども含め、運営者としてシートに行を追加します（→ [operator.md](operator.md)）。「棚をつくる」画面は shelf_config が空のときしか出ません。公開URLを知る第三者が棚を勝手に作れないようにするための制約です。

## 4. QRコード

棚ページのURL（`…/?s=（棚ID）`）をQR化して棚に貼る。QR生成は任意のツールで可（URLに個人情報は含まれないため、生成サービスの選定に神経質になる必要はありません）。

## 5. Code.gs を更新するとき

コードを直したら **デプロイ → デプロイを管理 → 編集（鉛筆）→ バージョン: 新バージョン → デプロイ**。これを忘れると古いコードが動き続けます（URLは変わりません）。
